コスト最適化の意図（テンプレートのコメントより）

NAT Gateway は 1a のみに配置しコストを抑制。DynamoDB と S3（ECR イメージレイヤー）は Gateway 型 VPC Endpoint 経由でアクセスし、NAT 課金を回避する設計になっている。逆に Secrets Manager／CloudWatch Logs／X-Ray／ECR API 呼び出し／外部 API（ipify.org）は Interface Endpoint が無いため NAT Gateway 経由で課金対象となる。

RDS は最小構成

db.t3.micro ・ Single-AZ ・ BackupRetentionPeriod: 0 のテスト用最小構成。マスターパスワードは RDS 管理シークレット（ManageMasterUserPassword）で自動生成・管理され、アプリは起動時に Secrets Manager から取得する。

CLAUDE.md との差異について

CLAUDE.md は「APM は後フェーズで追加、現フェーズでは未実装」としているが、実際の 04-ecs-alb.yaml には ADOT Java Agent 相当の OTEL 環境変数、CloudWatch Agent サイドカー（otel-collector）、X-Ray サンプリングルール、TaskRole への AWSXRayDaemonWriteAccess / CloudWatchAgentServerPolicy、Cpu/Memory 512/1024 への引き上げが、すでにすべて実装済みだった。ドキュメントが実装に追随していない可能性がある。

ECS タスクは AZ 間で流動的に配置

DesiredCount のデフォルトは 1 で、Private Subnet 1a/1c いずれかに配置される（図では両 AZ にまたがる形で 1 つのタスクとして表現）。ALB のリスナールールは /health・/configuration、/users*、/companies* をすべて同一ターゲットグループへフォワードする。
